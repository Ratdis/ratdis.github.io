    给MySQL添加配置文件my.cnf

    现在的新版MySQL都不会自带my.cnf之类的配置文件了…修改一些东西还挺麻烦的，贴一下个人已经配置好utf8mb4的my.cnf，存放在~/.my.cnf后重启MySQL即可

    将MySQL字符集编码改成utf8mb4

    如果你是直接使用上面我的my.cnf的话，就只需要进行最后一步修改Django中的setting.py

    1. 修改mysql配置文件，设置默认字符集utf8mb4, 包括collation
      [client]
      default-character-set = utf8mb4

      [mysql]
      default-character-set = utf8mb4

      [mysqld]
      character-set-client-handshake = FALSE
      character-set-server = utf8mb4
      collation-server = utf8mb4_unicode_ci
      init_connect=’SET NAMES utf8mb4’

    2. 重启，确认上述配置生效

      mysql> SHOW VARIABLES WHERE Variable_name LIKE ‘character_set_%’ OR Variable_name LIKE ‘collation%’;

      显示如下即配置成功

      +————————————-+——————————+
      | Variable_name | Value |
      +————————————-+——————————+
      | character_set_client | utf8mb4 |
      | character_set_connection | utf8mb4 |
      | character_set_database | utf8mb4 |
      | character_set_filesystem | binary |
      | character_set_results | utf8mb4 |
      | character_set_server | utf8mb4 |
      | character_set_system | utf8 |
      | collation_connection | utf8mb4_unicode_ci |
      | collation_database | utf8mb4_unicode_ci |
      | collation_server | utf8mb4_unicode_ci |
      +————————————-+——————————-+

    3. 在django项目的setting文件中,你必须设置charset=utf8mb4,
